다음 요구사항을 만족하는 완전한 구현 방법을 만들어줘.

[목표]
- Android Native PDF Viewer를 사용하여 PDF를 벡터 방식으로 렌더링
- 인터넷 없이 (오프라인) 동작
- 로컬 저장소에 있는 PDF 파일을 직접 열기
- 확대/축소(핀치 줌), 스크롤 지원
- 도면(PDF)을 확대해도 깨지지 않아야 함

[기술 조건]
- 기존 앱은 Kivy 기반 (Python)
- pyjnius를 사용해서 Android(Java) 코드와 연결
- Android 라이브러리는 Pdfium 기반 (예: AndroidPdfViewer)
- WebView는 사용하지 않음

[필수로 포함해줘]
1. Android PDF Viewer 라이브러리 사용 방법 (예: AndroidPdfViewer)
2. 필요한 Java 코드 (PDF View 생성 및 파일 로드)
3. Python 코드 (pyjnius로 Java View를 호출하는 코드)
4. Kivy UI 위에 Android View를 올리는 방법
5. PDF 파일 경로를 Android에서 읽을 수 있도록 처리하는 방법
6. buildozer.spec 설정 (매우 중요)
   - android.permissions
   - android.gradle_dependencies
   - 필요한 설정 전부 포함
7. 앱 실행 시 PDF를 여는 전체 흐름

[추가 요구사항]
- Kivy 화면은 유지하고, PDF 영역만 Native View로 표시
- 기존 버튼(UI)은 그대로 사용 가능해야 함
- 슬라이드(다음/이전 PDF 로딩) 확장 가능하도록 구조 설계

[중요]
- 실제로 APK 빌드 가능한 수준으로 작성
- 누락 없이 전체 코드 흐름을 제공
- 오류 발생 가능성이 있는 부분은 주석으로 설명














문제 상황:

* Python 코드에서는 `com.github.barteksc.pdfviewer.PDFView`를 사용하도록 구현 완료
* 하지만 APK 실행 시 PDF가 표시되지 않음 (강제종료됨)
* WebView 방식은 완전히 제거된 상태
* 원인은 buildozer.spec 설정 문제로 판단됨

현재 buildozer.spec의 requirements는 아래와 같음:
requirements = python3,kivy,openpyxl,et_xmlfile,jdcal,pyjnius,android,pysmb,pyasn1,six,tqdm

요구사항:

1. Native PDF Viewer가 정상 동작하도록 buildozer.spec을 수정하라
2. 반드시 필요한 android.gradle_dependencies를 추가하라
3. 불필요한 requirements는 정리하라 (최소 구성으로)
4. 수정된 buildozer.spec 전체를 완성된 형태로 출력하라
5. 왜 해당 설정이 필요한지도 간단히 설명하라

추가 조건:

* PDF는 인터넷 없이 오프라인에서 열려야 함
* WebView 또는 Google Docs Viewer는 절대 사용하지 않음
* 반드시 android-pdf-viewer 라이브러리를 사용해야 함
* Buildozer 기준으로 바로 빌드 가능한 상태여야 함
